## Output Format

Return valid JSON with exactly this structure:

{
  "entities": [
    {
      "name": "string — canonical name",
      "type": "ORGANIZATION | COUNTRY | LOCATION | PERSON | REGION | WEAPON_SYSTEM | MILITARY_UNIT | POLICY | TREATY | CONCEPT | VESSEL | AIRCRAFT | SATELLITE",
      "aliases": ["list of alternative names mentioned"],
      "confidence": 0.0-1.0
    }
  ],
  "relations": [
    {
      "source": "entity name (must match an entity above)",
      "target": "entity name (must match an entity above)",
      "type": "ALLIED_WITH | COMPETES_WITH | SANCTIONS | SUPPLIES_TO | OPERATES_IN | MEMBER_OF | COMMANDS | TARGETS | NEGOTIATES_WITH",
      "evidence": "brief quote or paraphrase from the source",
      "confidence": 0.0-1.0
    }
  ],
  "claims": [
    {
      "statement": "one clear declarative sentence",
      "type": "factual | assessment | prediction",
      "polarity": "positive | negative | neutral",
      "entities_involved": ["entity names referenced in this claim"],
      "confidence": 0.0-1.0,
      "temporal_scope": "time period this claim applies to (e.g. '2025', '2024-2026', 'ongoing')"
    }
  ]
}

## Rules

1. **Entities**: Use canonical names (e.g., "People's Republic of China" not "China" — but include "China" as alias). Merge co-referent mentions.
2. **Entity types — strict enforcement**: The `type` field MUST be exactly one of the 13 values listed above. Do NOT invent any other value. Forbidden examples: EVENT, TECHNOLOGY, FACILITY, OPERATION, INCIDENT, CONFLICT, CONCEPT_GROUP. If nothing fits, use CONCEPT as the fallback — never coin a new type.
3. **Events are NOT entities**: Operations, battles, attacks, exercises, conflicts, and incidents are NOT entities and must never appear in the `entities` array. Capture what happened, who was involved, and what the outcome was as CLAIMS instead. For example, an airstrike is a claim (factual), not an entity of type EVENT.
4. **Relations**: Only extract relations explicitly stated or strongly implied. Each must reference two entities from your entity list. Prefer the 9 listed relation types (ALLIED_WITH, COMPETES_WITH, SANCTIONS, SUPPLIES_TO, OPERATES_IN, MEMBER_OF, COMMANDS, TARGETS, NEGOTIATES_WITH); if none fits well, you may use a descriptive string — but prefer the canonical set first.
5. **Claims**: Distinguish factual (verifiable), assessment (analytical judgment), and prediction (future-oriented). One sentence per claim. No hedging — the confidence score captures uncertainty.
6. **Confidence**: 0.9+ = explicitly stated with evidence. 0.7-0.89 = strongly implied. 0.5-0.69 = weakly implied or ambiguous. Below 0.5 = don't extract.
7. **Temporal scope**: Always provide. Use "ongoing" if no time reference is given.
8. **Do NOT invent** information not present in the source. When in doubt, lower confidence rather than omit.
